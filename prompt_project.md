# Prompt Template Project Portfolio

Pakai prompt ini setiap kali mau bikin `project.md` baru untuk folder project lain.

## Prompt

Bantu saya menulis file `project.md` untuk portfolio pribadi saya sebagai Mobile Developer Flutter.

Tulis dalam bahasa Indonesia, ringkas, jelas, profesional, dan fokus ke portfolio. Jangan pakai bagian yang terlalu panjang, terlalu banyak saran, atau narasi interview. Hindari pengulangan.

Format wajib markdown dan ikuti struktur ini persis:

```md
# [Nama Project]

## Periode Project
[Contoh: Agustus 2025 - September 2025]

## Ringkasan Singkat
[1 paragraf singkat. Jelaskan project ini apa, untuk siapa, dan tujuan utamanya.]

## Problem yang Diselesaikan
- [masalah 1]
- [masalah 2]
- [masalah 3]

## Fitur Utama
- [fitur utama 1]
- [fitur utama 2]
- [fitur utama 3]
- [fitur utama 4]
- [fitur utama 5]

## Tech Stack
- [teknologi 1]
- [teknologi 2]
- [teknologi 3]
- [teknologi 4]

## Repository
- GitHub: [link repository]
- Play Store: [link Play Store, opsional]

## Arsitektur Project
[1 paragraf singkat tentang pendekatan arsitektur, pembagian module, atau alasan struktur codebase.]

## Struktur Project
```text
[tree folder utama project]
```

## Tantangan Teknis
### [Tantangan 1]
[1 paragraf singkat.]

### [Tantangan 2]
[1 paragraf singkat.]

### [Tantangan 3]
[1 paragraf singkat.]

## Impact
- [impact 1]
- [impact 2]
- [impact 3]
```

## Aturan Isi

- Fokus pada hasil dan engineering value.
- Maksimal 1 paragraf untuk tiap section naratif.
- Gunakan bullet untuk section yang memang list.
- Jika project punya struktur folder yang menarik atau relevan, tulis dalam block code `text`.
- Jangan buat section seperti:
  - "Versi Singkat untuk Portofolio"
  - "Versi Menengah untuk CV atau LinkedIn"
  - "Poin yang Sebaiknya Dipakai Saat Presentasi"
  - "Contoh Narasi Siap Pakai"
  - "Kalau Mau Diposisikan Sebagai Hasil Belajar"
  - "Ringkasan Akhir"
- Jangan tambahkan saran presentasi, saran interview, atau kata kunci SEO.
- Jangan terlalu marketing, jangan terlalu akademik.
- Nada tulisan harus terasa seperti portfolio engineer yang rapi dan to the point.

## Input yang Akan Saya Berikan

Saya biasanya akan memberi:

- nama project
- periode project
- tujuan project
- problem yang diselesaikan
- fitur utama
- stack yang dipakai
- link GitHub repository
- link Play Store jika ada
- arsitektur atau pola yang dipakai
- struktur folder project jika relevan
- tantangan teknis
- impact atau hasil

Kalau ada informasi yang belum lengkap, tetap bantu susun versi terbaik tanpa membuat bagian jadi terlalu panjang.
